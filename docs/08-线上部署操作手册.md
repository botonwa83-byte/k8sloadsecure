# 线上部署操作手册

## 前置条件

- CentOS 7 服务器上已安装 Docker >= 20.10
- 可访问镜像仓库（如 registry.wanfangdata.com.cn）
- 可执行 kubectl 命令操作 K8s 集群
- MySQL k8sgate 数据库已创建（init.sql 已执行）

## 第一步：拉取代码

```bash
cd /opt
git clone https://github.com/botonwa83-byte/k8sloadsecure.git
cd k8sloadsecure
```

## 第二步：安装编译环境

### 安装 Go（如未安装）

```bash
wget https://go.dev/dl/go1.22.5.linux-amd64.tar.gz
tar -C /usr/local -xzf go1.22.5.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> /etc/profile
source /etc/profile
go version
```

### 安装 Node.js（如未安装）

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
yum install -y nodejs
node --version
npm --version
```

## 第三步：构建 Docker 镜像

### 方式一：使用 Makefile（推荐）

```bash
# 修改镜像仓库地址（如果不是默认的）
# 默认为 registry.wanfangdata.com.cn/k8sgate

# 构建镜像
make docker-all

# 打版本号标签（可选）
docker tag registry.wanfangdata.com.cn/k8sgate/backend:latest \
           registry.wanfangdata.com.cn/k8sgate/backend:v0.3.0
docker tag registry.wanfangdata.com.cn/k8sgate/frontend:latest \
           registry.wanfangdata.com.cn/k8sgate/frontend:v0.3.0

# 推送到镜像仓库
make push-all
```

### 方式二：手动构建

```bash
# 构建后端镜像
cd backend
docker build -t registry.wanfangdata.com.cn/k8sgate/backend:latest .
cd ..

# 构建前端镜像
cd frontend
npm install
docker build -t registry.wanfangdata.com.cn/k8sgate/frontend:latest .
cd ..

# 推送
docker push registry.wanfangdata.com.cn/k8sgate/backend:latest
docker push registry.wanfangdata.com.cn/k8sgate/frontend:latest
```

## 第四步：配置 Secret

```bash
cd deploy

# 复制 Secret 模板
cp secret.yaml.example secret.yaml

# 编辑 secret.yaml，填入真实的数据库地址、用户名、密码和 JWT 密钥
vi secret.yaml
```

secret.yaml 中需要修改的字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| host | MySQL 主节点地址 | 10.0.1.100 |
| username | MySQL 用户名 | k8sgate |
| password | MySQL 密码 | YourStrongPassword |
| secret | JWT 签名密钥（随机字符串） | 用 `openssl rand -base64 32` 生成 |

## 第五步：部署到 K8s

```bash
# 方式一：一键部署
make deploy

# 方式二：分步部署（便于排查问题）
kubectl apply -f deploy/namespace.yaml
kubectl apply -f deploy/rbac.yaml
kubectl apply -f deploy/secret.yaml
kubectl apply -f deploy/backend.yaml
kubectl apply -f deploy/frontend.yaml
kubectl apply -f deploy/virtualservice.yaml
```

## 第六步：确认 Istio 网关名称

部署前需要确认你们实际使用的 Istio Gateway 名称：

```bash
# 查看现有 Gateway
kubectl get gateway -A
```

然后修改 `deploy/virtualservice.yaml` 中的 `gateways` 字段为实际值：

```yaml
gateways:
  - istio-system/你的实际网关名称
```

## 第七步：验证部署

```bash
# 查看 Pod 状态
kubectl get pods -n k8sgate-system

# 查看日志
kubectl logs -n k8sgate-system -l app=k8sgate-backend --tail=50

# 查看 Service
kubectl get svc -n k8sgate-system
```

期望输出：
```
NAME               READY   STATUS    RESTARTS   AGE
k8sgate-backend-xxx   1/1   Running   0          1m
k8sgate-backend-xxx   1/1   Running   0          1m
k8sgate-frontend-xxx  1/1   Running   0          1m
k8sgate-frontend-xxx  1/1   Running   0          1m
```

## 第八步：配置 DNS

将域名 `k8sgate.wanfangdata.com.cn` 解析到 Istio IngressGateway 的外部 IP：

```bash
# 查看 IngressGateway 的 IP
kubectl get svc -n istio-system istio-ingressgateway
```

在 DNS 服务器或 /etc/hosts 中添加解析记录。

## 第九步：访问系统

浏览器打开 `http://k8sgate.wanfangdata.com.cn`

默认管理员账号：
- 用户名：`admin`
- 密码：`Admin@123`

## 常见问题

### 后端 Pod 启动失败

```bash
# 查看详细日志
kubectl logs -n k8sgate-system deploy/k8sgate-backend

# 常见原因：
# 1. 数据库连接失败 → 检查 Secret 中的数据库配置
# 2. K8s API 权限不足 → 检查 RBAC 配置是否 apply
```

### 前端页面白屏

```bash
# 检查前端 Pod 日志
kubectl logs -n k8sgate-system deploy/k8sgate-frontend

# 常见原因：
# 1. 后端 Service 名称不匹配 → 检查 nginx.conf 中的 proxy_pass
# 2. API 路径不对 → 检查浏览器开发者工具 Network 选项卡
```

### VirtualService 不生效

```bash
# 确认 Gateway 名称
kubectl get gateway -A
# 修改 virtualservice.yaml 中的 gateways 字段
```

## 版本更新流程

```bash
# 1. 拉取最新代码
cd /opt/k8sloadsecure
git pull

# 2. 重新构建镜像
make docker-all

# 3. 打新版本标签
docker tag registry.wanfangdata.com.cn/k8sgate/backend:latest \
           registry.wanfangdata.com.cn/k8sgate/backend:vX.Y.Z

# 4. 推送镜像
make push-all

# 5. 滚动更新
kubectl rollout restart deployment/k8sgate-backend -n k8sgate-system
kubectl rollout restart deployment/k8sgate-frontend -n k8sgate-system

# 6. 查看更新状态
kubectl rollout status deployment/k8sgate-backend -n k8sgate-system
```
