# 部署方案

## 1. 系统命名空间

在集群中创建独立命名空间 `k8sgate-system`，所有系统资源部署在此命名空间下。

```bash
kubectl create namespace k8sgate-system
```

## 2. 组件清单

| 组件 | 类型 | 副本数 | 说明 |
|------|------|--------|------|
| k8sgate-backend | Deployment | 2 | Go 后端服务 |
| k8sgate-frontend | Deployment | 2 | Nginx 托管 Vue 静态文件 |
| k8sgate-backend-svc | Service (ClusterIP) | - | 后端服务 |
| k8sgate-frontend-svc | Service (ClusterIP) | - | 前端服务 |
| k8sgate-vs | VirtualService | - | Istio 流量入口 |
| k8sgate-gw | Gateway | - | Istio 网关（如复用现有网关则不需要） |
| k8sgate-admin-sa | ServiceAccount | - | 管理员用 ServiceAccount |

## 3. 后端 Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: k8sgate-backend
  namespace: k8sgate-system
spec:
  replicas: 2
  selector:
    matchLabels:
      app: k8sgate-backend
  template:
    metadata:
      labels:
        app: k8sgate-backend
    spec:
      serviceAccountName: k8sgate-controller
      containers:
        - name: backend
          image: registry.wanfangdata.com.cn/k8sgate/backend:latest
          ports:
            - containerPort: 8080
          env:
            - name: DB_HOST
              valueFrom:
                secretKeyRef:
                  name: k8sgate-db-secret
                  key: host
            - name: DB_PORT
              value: "3306"
            - name: DB_USER
              valueFrom:
                secretKeyRef:
                  name: k8sgate-db-secret
                  key: username
            - name: DB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: k8sgate-db-secret
                  key: password
            - name: DB_NAME
              value: "k8sgate"
            - name: JWT_SECRET
              valueFrom:
                secretKeyRef:
                  name: k8sgate-jwt-secret
                  key: secret
            - name: DASHBOARD_URL
              value: "https://kubernetes-dashboard.kubernetes-dashboard.svc"
          resources:
            requests:
              cpu: 100m
              memory: 128Mi
            limits:
              cpu: 500m
              memory: 512Mi
          livenessProbe:
            httpGet:
              path: /healthz
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 10
          readinessProbe:
            httpGet:
              path: /healthz
              port: 8080
            initialDelaySeconds: 3
            periodSeconds: 5
```

## 6. Istio VirtualService

```yaml
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: k8sgate
  namespace: k8sgate-system
spec:
  hosts:
    - "k8sgate.wanfangdata.com.cn"
  gateways:
    - istio-system/default-gateway  # 复用现有网关，具体名称根据实际调整
  http:
    - match:
        - uri:
            prefix: /api/
      route:
        - destination:
            host: k8sgate-backend
            port:
              number: 8080
    - match:
        - uri:
            prefix: /dashboard/
      route:
        - destination:
            host: k8sgate-backend
            port:
              number: 8080
    - route:
        - destination:
            host: k8sgate-frontend
            port:
              number: 80
```

## 7. RBAC 配置

后端需要一个高权限的 ServiceAccount 用于管理其他 ServiceAccount 和 RoleBinding：

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: k8sgate-controller
  namespace: k8sgate-system

---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: k8sgate-controller
subjects:
  - kind: ServiceAccount
    name: k8sgate-controller
    namespace: k8sgate-system
roleRef:
  kind: ClusterRole
  name: cluster-admin
  apiGroup: rbac.authorization.k8s.io
```

## 8. Secret 配置

部署前需创建以下 Secret：

```bash
# 数据库连接信息
kubectl create secret generic k8sgate-db-secret \
  --namespace k8sgate-system \
  --from-literal=host=mysql-master.example.com \
  --from-literal=username=k8sgate \
  --from-literal=password=YOUR_DB_PASSWORD

# JWT 签名密钥
kubectl create secret generic k8sgate-jwt-secret \
  --namespace k8sgate-system \
  --from-literal=secret=$(openssl rand -base64 32)
```

## 9. 构建与发布

### Dockerfile（后端）

```dockerfile
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o k8sgate-backend ./main.go

FROM alpine:3.19
RUN apk add --no-cache ca-certificates tzdata
ENV TZ=Asia/Shanghai
COPY --from=builder /app/k8sgate-backend /usr/local/bin/
EXPOSE 8080
ENTRYPOINT ["k8sgate-backend"]
```

### Dockerfile（前端）

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:1.25-alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

## 10. 部署步骤

```bash
# 1. 创建命名空间
kubectl create namespace k8sgate-system

# 2. 创建 Secret
kubectl create secret generic k8sgate-db-secret --namespace k8sgate-system ...
kubectl create secret generic k8sgate-jwt-secret --namespace k8sgate-system ...

# 3. 创建 RBAC
kubectl apply -f deploy/rbac.yaml

# 4. 初始化数据库（连接 MySQL 执行建表 SQL）
mysql -h mysql-master -u k8sgate -p k8sgate < deploy/init.sql

# 5. 部署后端
kubectl apply -f deploy/backend.yaml

# 6. 部署前端
kubectl apply -f deploy/frontend.yaml

# 7. 配置 Istio 路由
kubectl apply -f deploy/virtualservice.yaml

# 8. 配置 DNS 解析
# 将 k8sgate.wanfangdata.com.cn 解析到 Istio IngressGateway 的外部 IP
```
