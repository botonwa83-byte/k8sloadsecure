# K8sGate 部署方案

> 本文档为唯一有效的部署指南，适用于测试环境和生产环境。

## 1. 架构概览

```
开发机 (macOS)                          服务器 (Rocky Linux 9.x)
┌─────────────────────┐                ┌──────────────────────────────┐
│                     │                │  systemd                     │
│  Go 源码            │   make         │  ┌─────────────────────┐     │
│  Vue 源码           │  deploy-test   │  │ k8sgate-backend     │     │
│  deploy/ 脚本       │ ────────────▶  │  │ (Go 二进制, :8080)  │     │
│                     │   scp + ssh    │  └────────┬────────────┘     │
│  交叉编译:          │                │           │                  │
│  linux/amd64        │                │  ┌────────▼────────────┐     │
│                     │                │  │ k8sgate-frontend    │     │
└─────────────────────┘                │  │ (podman nginx, :80) │     │
                                       │  └────────┬────────────┘     │
                                       │           │                  │
                                       │  ┌────────▼────────────┐     │
                                       │  │ MySQL 5.7 (:3306)   │     │
                                       │  └─────────────────────┘     │
                                       │           │                  │
                                       │  ┌────────▼────────────┐     │
                                       │  │ K8s Dashboard       │     │
                                       │  │ (ClusterIP :443)    │     │
                                       │  └─────────────────────┘     │
                                       └──────────────────────────────┘
```

**关键决策**：
- 后端以 systemd 服务运行（非容器化），直接使用节点上的 kubeconfig
- 前端以 podman 容器运行（nginx），使用 `--network=host` 访问后端
- K8s Dashboard 为 ClusterIP 类型，只能通过后端代理访问

## 2. 环境要求

### 开发机（构建端）

| 组件 | 版本 | 用途 |
|------|------|------|
| Go | 1.21+ | 交叉编译后端 |
| Node.js | 18+ | 构建前端 |
| make | - | 构建命令入口 |

### 服务器（运行端）

| 组件 | 版本 | 用途 |
|------|------|------|
| OS | Rocky Linux 9.x | - |
| podman | 5.x | 运行 nginx 前端容器 |
| kubectl | 1.30+ | 操作 K8s 集群 |
| MySQL | 5.7 | 数据存储 |
| systemd | - | 服务管理 |

> **注意**：服务器上不需要 Go、Node.js、Docker。所有编译在开发机完成。

## 3. 部署流程

### 3.1 一键部署（推荐）

在开发机上执行：

```bash
# 首次部署需要配置 SSH 免密登录（可选）
ssh-copy-id root@10.10.184.243

# 一键构建 + 上传 + 部署
make deploy-test
```

`make deploy-test` 依次执行：
1. 交叉编译后端 → `backend/k8sgate-backend`（linux/amd64 ELF）
2. 构建前端 → `frontend/dist/`
3. 打包 → `release/k8sloadsecure-<version>-<commit>-<date>.tar.gz`
4. SCP 上传到服务器
5. SSH 执行 `deploy-test.sh`（初始化数据库、替换文件、重启服务）

### 3.2 分步部署

如果需要更精细的控制：

```bash
# 步骤 1：构建打包（在开发机）
make package
# 产物：release/k8sloadsecure-latest.tar.gz

# 步骤 2：上传（在开发机）
scp release/k8sloadsecure-latest.tar.gz root@10.10.184.243:/tmp/

# 步骤 3：在服务器上解压和部署
ssh root@10.10.184.243
mkdir -p /opt/k8sgate-deploy
tar -xzf /tmp/k8sloadsecure-latest.tar.gz -C /opt/k8sgate-deploy --strip-components=1
cd /opt/k8sgate-deploy
bash deploy/deploy-test.sh
```

### 3.3 仅更新后端（热修复）

修改代码后，只替换后端二进制：

```bash
# 在开发机上
cd backend
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags="-s -w" -o k8sgate-backend .

# 上传并重启
scp k8sgate-backend root@10.10.184.243:/tmp/
ssh root@10.10.184.243 'systemctl stop k8sgate-backend && \
  cp -f /tmp/k8sgate-backend /opt/k8sgate/k8sgate-backend && \
  chmod +x /opt/k8sgate/k8sgate-backend && \
  systemctl start k8sgate-backend'
```

### 3.4 仅更新前端

```bash
# 在开发机上
cd frontend && npm run build

# 上传并重启
scp -r dist root@10.10.184.243:/opt/k8sgate/dist
ssh root@10.10.184.243 'systemctl restart k8sgate-frontend'
```

## 4. 构建系统说明

### 4.1 package.sh 做了什么

```
源码 ──▶ 编译（交叉编译 + npm build）──▶ 打包（tar.gz）
```

- 后端：`CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags="-s -w"`
- 前端：`npm run build`
- 打包内容：只包含编译产物（二进制 + dist），不包含源码
- 包名含 git commit：`k8sloadsecure-1.0.0-bfb09d8-20260526_110319.tar.gz`
- 包内有 `VERSION` 文件记录版本信息
- 内置架构校验：确保产物是 linux ELF 64-bit

### 4.2 deploy-test.sh 做了什么

```
解压包 ──▶ 校验 ──▶ 初始化DB ──▶ 停服务 ──▶ 替换文件 ──▶ 启服务 ──▶ 健康检查
```

1. 环境检测（MySQL、kubectl、Dashboard）
2. 验证构建产物（ELF 架构校验）
3. 初始化数据库（幂等执行 init.sql）
4. 停旧服务 → 拷贝新文件 → 启新服务
5. 自动检测 Dashboard URL 并写入 .env
6. 健康检查验证

### 4.3 Makefile 命令速查

| 命令 | 说明 |
|------|------|
| `make build-backend` | 交叉编译后端 |
| `make build-frontend` | 构建前端 |
| `make build-all` | 编译全部 |
| `make package` | 编译 + 打包为 tar.gz |
| `make deploy-test` | 编译 + 打包 + 上传 + 部署 |
| `make clean` | 清理编译产物 |
| `make dev-backend` | 本地开发启动后端 |
| `make dev-frontend` | 本地开发启动前端 |

## 5. 服务器端配置

### 5.1 服务管理

```bash
# 查看状态
systemctl status k8sgate-backend
systemctl status k8sgate-frontend

# 重启
systemctl restart k8sgate-backend
systemctl restart k8sgate-frontend

# 查看日志（实时）
journalctl -u k8sgate-backend -f
journalctl -u k8sgate-frontend -f

# 查看前端容器日志
podman logs k8sgate-frontend
```

### 5.2 部署目录结构

```
/opt/k8sgate/
├── k8sgate-backend     # Go 二进制
├── dist/               # 前端静态文件
│   ├── index.html
│   └── assets/
├── nginx.conf          # nginx 配置
├── .env                # 运行时环境变量
└── VERSION             # 版本信息
```

### 5.3 环境变量（.env）

部署脚本自动生成，一般无需手动修改：

| 变量 | 说明 | 示例 |
|------|------|------|
| DB_HOST | 数据库地址 | 127.0.0.1 |
| DB_PORT | 数据库端口 | 3306 |
| DB_USER | 数据库用户 | root |
| DB_PASSWORD | 数据库密码 | Admin@123 |
| DB_NAME | 数据库名 | k8sgate_test |
| JWT_SECRET | JWT 签名密钥 | k8sgate-test-secret-2026 |
| DASHBOARD_URL | Dashboard 服务地址 | https://10.98.173.203:443 |
| SERVER_PORT | 后端端口 | 8080 |
| PASSWORD_MAX_AGE | 密码有效期（天） | 90 |
| GIN_MODE | Gin 运行模式 | release |

### 5.4 Dashboard 配置

部署脚本自动处理以下配置：
- 检测 Dashboard ClusterIP 地址
- 创建 ClusterRoleBinding（k8sgate-dashboard-admin）
- 启用 `--enable-skip-login`（Dashboard 使用自身 SA 的 cluster-admin 权限）
- 访问控制由 K8sGate 代理层实现

## 6. 验证部署

```bash
# 后端健康检查
curl http://10.10.184.243:8080/healthz
# 预期：{"status":"ok"}

# 前端页面
curl -s http://10.10.184.243/ | head -5
# 预期：<!doctype html>...

# 查看部署版本
cat /opt/k8sgate/VERSION

# Dashboard 连通性
curl -sk $(grep DASHBOARD_URL /opt/k8sgate/.env | cut -d= -f2) -o /dev/null -w 'HTTP %{http_code}\n'
# 预期：HTTP 200
```

浏览器访问：`http://10.10.184.243`，默认管理员 `admin / Admin@123`

## 7. 故障排除

| 问题 | 排查命令 | 常见原因 |
|------|----------|----------|
| 后端启动失败 | `journalctl -u k8sgate-backend -n 20` | 数据库连接失败、端口占用 |
| 前端白屏 | `podman logs k8sgate-frontend` | nginx 配置错误、dist 文件缺失 |
| Dashboard 不可用 | `curl -sk $DASHBOARD_URL` | Dashboard Pod 未运行、URL 配置错误 |
| API 404 | 对比服务器和本地的 `md5sum k8sgate-backend` | 部署了旧版二进制（未重新编译）|
| 登录失败 | 检查数据库 `SELECT * FROM users;` | 数据库未初始化、密码过期 |

## 8. 生产环境注意事项

测试环境部署方案可直接用于生产，但需要调整：

- [ ] 修改 JWT_SECRET 为强随机密钥（`openssl rand -base64 32`）
- [ ] 修改数据库密码
- [ ] 配置 TLS（nginx 启用 HTTPS）
- [ ] 配置防火墙（只开放 80/443）
- [ ] Dashboard 设为 ClusterIP + NetworkPolicy（禁止绕过代理直接访问）
- [ ] 配置日志轮转
- [ ] 配置定期数据库备份
