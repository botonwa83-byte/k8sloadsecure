## 2. 数据库配置

### 2.1 创建数据库

```sql
CREATE DATABASE k8sgate 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

CREATE USER 'k8sgate'@'%' 
  IDENTIFIED WITH mysql_native_password BY 'your_strong_password';

GRANT ALL PRIVILEGES ON k8sgate.* TO 'k8sgate'@'%';
FLUSH PRIVILEGES;
```

### 2.2 MySQL 5.7 特殊配置

```ini
# /etc/my.cnf
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
default-storage-engine=InnoDB
innodb_buffer_pool_size=4G
max_connections=1000
sql_mode=STRICT_TRANS_TABLES,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION
```

### 2.3 导入初始化数据

```bash
mysql -u k8sgate -p k8sgate < deploy/init.sql
```

## 3. 后端部署

### 3.1 配置文件

创建 `backend/.env` 文件：

```ini
# 服务配置
SERVER_PORT=8080
GIN_MODE=release

# 数据库配置
DB_HOST=your-db-host
DB_PORT=3306
DB_USER=k8sgate
DB_PASSWORD=your_strong_password
DB_NAME=k8sgate
DB_MAX_OPEN_CONNS=20
DB_MAX_IDLE_CONNS=10

# JWT 配置
JWT_SECRET=your_32_characters_secret_key_here
TOKEN_EXPIRE_HOURS=24

# K8s 配置
KUBE_CONFIG_PATH=/root/.kube/config
DASHBOARD_URL=https://kubernetes-dashboard.kubernetes-dashboard.svc

# 日志配置
LOG_LEVEL=info
LOG_FORMAT=json
```

### 3.2 编译部署

```bash
# 进入后端目录
cd backend

# 编译
go build -o k8sgate .

# 创建服务用户
useradd -r -s /sbin/nologin k8sgate

# 复制文件
cp k8sgate /usr/local/bin/
mkdir -p /etc/k8sgate
cp .env /etc/k8sgate/
mkdir -p /var/log/k8sgate
chown -R k8sgate:k8sgate /var/log/k8sgate
```

### 3.3 Systemd 服务配置

创建 `/etc/systemd/system/k8sgate.service`：

```ini
[Unit]
Description=K8sLoadSecure API Service
After=network.target mysqld.service

[Service]
User=k8sgate
Group=k8sgate
WorkingDirectory=/usr/local/bin
ExecStart=/usr/local/bin/k8sgate
Restart=always
RestartSec=5
Environment=GIN_MODE=release

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
systemctl daemon-reload
systemctl enable k8sgate
systemctl start k8sgate
systemctl status k8sgate
```

## 4. 前端部署

### 4.1 构建

```bash
cd frontend

# 安装依赖
npm install --production

# 构建生产版本
npm run build

# 复制到静态目录
mkdir -p /var/www/k8sgate
cp -r dist/* /var/www/k8sgate/
```

### 4.2 Nginx 配置

创建 `/etc/nginx/conf.d/k8sgate.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/k8sgate;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Dashboard 代理
    location /k8s-dashboard/ {
        proxy_pass http://localhost:8080/dashboard/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Authorization "";
    }

    # 健康检查
    location /healthz {
        proxy_pass http://localhost:8080/healthz;
    }
}
```

重启 Nginx：

```bash
systemctl reload nginx
```

## 5. K8s Dashboard 配置

### 5.1 安装 Dashboard

```bash
kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml
```

### 5.2 配置跳过登录

```bash
kubectl patch deployment kubernetes-dashboard \
  -n kubernetes-dashboard \
  --type='json' \
  -p='[{"op": "add", "path": "/spec/template/spec/containers/0/args/-", "value": "--enable-skip-login"}]'

kubectl patch deployment kubernetes-dashboard \
  -n kubernetes-dashboard \
  --type='json' \
  -p='[{"op": "add", "path": "/spec/template/spec/containers/0/args/-", "value": "--disable-settings-authorizer"}]'
```

### 5.3 创建管理员权限

```bash
kubectl apply -f - <<EOF
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: kubernetes-dashboard-admin
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: kubernetes-dashboard
  namespace: kubernetes-dashboard
EOF
```

### 5.4 安全加固（防止绕过权限管理）

#### 5.4.1 修改为 ClusterIP（禁止外部直接访问）

```bash
kubectl patch service kubernetes-dashboard \
  -n kubernetes-dashboard \
  -p '{"spec":{"type":"ClusterIP"}}'
```

#### 5.4.2 应用网络隔离策略

```bash
kubectl apply -f deploy/dashboard-network-policy.yaml
```

**`deploy/dashboard-network-policy.yaml`** 内容：

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: restrict-dashboard
  namespace: kubernetes-dashboard
spec:
  podSelector:
    matchLabels:
      k8s-app: kubernetes-dashboard
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: k8sgate
    ports:
    - protocol: TCP
      port: 8443
```

#### 5.4.3 应用完整安全配置

```bash
kubectl apply -f deploy/dashboard-secure-config.yaml
```

#### 5.4.4 安全加固效果

| 行为 | 加固前 | 加固后 |
|------|--------|--------|
| 直接访问 Dashboard | ✅ 可访问 | ❌ 拒绝 |
| 通过 k8sgate 访问 | ✅ 可访问 | ✅ 可访问 |
| 绕过权限系统 | ✅ 可能 | ❌ 不可能 |

## 6. 安全配置

### 6.1 防火墙配置

```bash
# 开放 HTTP/HTTPS 端口
firewall-cmd --add-port=80/tcp --permanent
firewall-cmd --add-port=443/tcp --permanent
firewall-cmd --reload

# 限制数据库访问
firewall-cmd --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="3306" protocol="tcp" accept' --permanent
```

### 6.2 TLS 配置

```bash
# 安装 Certbot
yum install certbot python3-certbot-nginx -y

# 获取证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

### 6.3 安全 Headers

在 Nginx 配置中添加：

```nginx
add_header X-Frame-Options "DENY";
add_header X-XSS-Protection "1; mode=block";
add_header X-Content-Type-Options "nosniff";
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";
```

## 7. 监控与日志

### 7.1 日志轮转

创建 `/etc/logrotate.d/k8sgate`：

```ini
/var/log/k8sgate/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 640 k8sgate k8sgate
    postrotate
        systemctl kill -s USR1 k8sgate
    endscript
}
```

### 7.2 Prometheus 监控

添加 metrics 端点到后端：

```go
// main.go
import "github.com/prometheus/client_golang/prometheus/promhttp"

r.GET("/metrics", gin.WrapH(promhttp.Handler()))
```

## 附录：故障排除

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据库连接失败 | 认证方式不兼容 | 修改为 `mysql_native_password` |
| Dashboard 403 | 权限不足 | 创建 ClusterRoleBinding |
| 前端空白页 | 静态资源路径错误 | 检查 Nginx root 配置 |
| 服务无法启动 | 配置文件错误 | 检查 `.env` 文件格式 |

| 服务 | 日志路径 |
|------|----------|
| 后端 | `/var/log/k8sgate/app.log` |
| Nginx | `/var/log/nginx/access.log` |
| MySQL | `/var/log/mysqld.log` |
