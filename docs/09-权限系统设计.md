## 1. 当前权限系统分析

### 1.1 当前角色体系

| 角色 | 权限范围 | 说明 |
|------|----------|------|
| admin | 全部权限 | 集群管理员 |
| global_viewer | 全局只读 | 可查看所有资源 |
| developer | 项目权限 | 只能访问分配的项目 |

### 1.2 当前项目权限

| 权限 | 说明 |
|------|------|
| read | 只读权限 |
| readwrite | 读写权限 |

### 1.3 存在的问题

| 问题 | 描述 |
|------|------|
| 权限粒度太粗 | 只有两种权限级别 |
| 角色不够灵活 | 无法自定义角色 |
| 资源类型未细分 | 所有资源使用同一权限 |
| 缺少特殊角色 | 如安全管理员、运维管理员等 |

## 2. 需求分析

### 2.1 功能需求

| 需求 | 描述 |
|------|------|
| 细粒度权限 | 不同资源类型独立权限 |
| 自定义角色 | 支持创建自定义角色 |
| 项目级权限 | 针对特定项目的权限配置 |
| 角色继承 | 角色可以继承其他角色权限 |
| 临时权限 | 支持限时权限 |
| 审批流程 | 权限变更需要审批 |

### 2.2 角色需求

| 角色类型 | 描述 |
|----------|------|
| 系统角色 | 内置角色，不可修改 |
| 自定义角色 | 用户创建的角色 |
| 组合角色 | 多个角色组合 |

## 8. 实现方案

### 8.1 权限判定逻辑

```go
func CheckPermission(userID uint, resource string, action string) bool {
    // 获取用户所有角色
    roles := getUserRoles(userID)
    
    // 获取角色权限
    permissions := getRolePermissions(roles)
    
    // 检查权限
    for _, p := range permissions {
        if p.Resource == resource && contains(p.Actions, action) {
            return true
        }
    }
    
    return false
}
```

### 8.2 项目级权限判定

```go
func CheckProjectPermission(userID uint, projectID uint, resource string, action string) bool {
    // 检查系统角色权限
    if CheckPermission(userID, resource, action) {
        return true
    }
    
    // 检查项目角色权限
    projectRoles := getUserProjectRoles(userID, projectID)
    permissions := getRolePermissions(projectRoles)
    
    for _, p := range permissions {
        if p.Resource == resource && contains(p.Actions, action) {
            return true
        }
    }
    
    return false
}
```

### 8.3 中间件实现

```go
func PermissionMiddleware(resource string, action string) gin.HandlerFunc {
    return func(c *gin.Context) {
        userID := middleware.GetUserID(c)
        
        if !CheckPermission(userID, resource, action) {
            pkg.Fail(c, http.StatusForbidden, 40301, "权限不足")
            c.Abort()
            return
        }
        
        c.Next()
    }
}
```

UPDATE user_roles SET role_id = (SELECT id FROM roles WHERE name = 'admin') WHERE role = 'admin';

## 10. 安全考虑

| 安全措施 | 描述 |
|----------|------|
| 最小权限原则 | 用户只获得必要的最小权限 |
| 权限审计 | 记录所有权限变更 |
| 定期复核 | 定期检查权限分配 |
| 权限过期 | 临时权限自动过期 |
| 双因素认证 | 敏感操作需要额外验证 |
